# EDA HoReCa Moscow

Исследовательский анализ заведений общественного питания Москвы.

## О проекте
Проект посвящен исследовательскому анализу рынка общественного питания Москвы: подготовке данных, проверке гипотез и формированию практических рекомендаций для открытия нового заведения.

Основной файл проекта: `EDA_HoReCa_Moscow.ipynb`.

## Данные
Источник данных:
- `https://code.s3.yandex.net/datasets/rest_info.csv`
- `https://code.s3.yandex.net/datasets/rest_price.csv`

В датасетах содержатся:
- название и категория заведения;
- адрес и административный округ;
- статус сетевого заведения;
- количество посадочных мест;
- режим работы;
- рейтинг;
- ценовая категория и показатели среднего чека.

## Что сделано
1. Объединены таблицы `rest_info` и `rest_price` в единый датафрейм.
2. Приведены типы данных:
- `seats` -> `Int16` (через nullable `Int64`);
- `chain` -> `int8`;
- `rating` -> `float32`.
3. Обработаны пропуски:
- `price` заполнен значением `неопределено`;
- `hours` заполнен значением `нет_данных`;
- для остальных признаков сохранен исходный подход с учетом аналитической задачи.
4. Нормализованы `name` и `address`, затем удалены дубликаты по ключевым полям (`name`, `address`).
5. Создан признак `is_24_7` по наличию признака круглосуточной работы в `hours`.
6. Выполнен EDA:
- структура категорий и округов;
- анализ сетевых и несетевых форматов;
- распределения `seats`, `rating`, `middle_avg_bill`;
- корреляционный анализ (в том числе `phi_k`);
- топ-15 сетей Москвы по числу точек и их средним рейтингам.
7. Сформированы итоговые выводы и рекомендации для бизнес-задачи.

## Как запустить
1. Создать виртуальное окружение и активировать его:
```bash
python3 -m venv .venv
source .venv/bin/activate
```
2. Установить зависимости:
```bash
pip install -r requirements.txt
```
3. Запустить Jupyter:
```bash
jupyter notebook
```
4. Открыть `EDA_HoReCa_Moscow.ipynb` и выполнить ячейки сверху вниз.
